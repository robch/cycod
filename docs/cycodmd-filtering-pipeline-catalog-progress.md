## Documentation Quality Standards

Each pair of files (catalog + proof) follows this pattern:

### Catalog Document (`*-layer-N.md`)
- **Purpose**: User-facing documentation
- **Contents**:
  - Layer purpose and role in pipeline
  - All command-line options
  - Syntax and examples
  - Data flow description
  - Integration with other layers
  - Common use cases

### Proof Document (`*-layer-N-proof.md`)
- **Purpose**: Technical evidence and source code references
- **Contents**:
  - Exact line numbers for parser implementation
  - Code snippets showing option handling
  - Command property declarations
  - Execution flow with line numbers
  - Helper method implementations
  - Data flow with code evidence
  - Error handling proof
  - Integration points with code references
