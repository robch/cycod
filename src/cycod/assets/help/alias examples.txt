CYCOD ALIAS EXAMPLES

  Aliases save frequently used command-line options for reuse.
  Create specialized workflows for development, analysis, and automation tasks.

DEVELOPMENT WORKFLOW ALIASES

  EXAMPLE 1: Create a React development alias

    cycod alias add react-dev "--add-system-prompt \"You are a React expert focusing on modern hooks, TypeScript, and performance optimization\" --auto-approve read write"

  EXAMPLE 2: Create a code review alias

    cycod alias add code-review "--add-user-prompt \"Review this code for bugs, security issues, and best practices. Provide specific examples and fixes.\" --auto-approve read" --user

  EXAMPLE 3: Create a DevOps troubleshooting alias

    cycod alias add devops-debug "--add-system-prompt \"You are a senior DevOps engineer specializing in Kubernetes, Docker, and CI/CD pipelines\" --auto-approve read" --global

  EXAMPLE 15: Use alias with input files

ALIAS MANAGEMENT

  EXAMPLE 16: List all available aliases

    cycod alias list

  EXAMPLE 17: View specific alias definition

    cycod alias get react-dev

  EXAMPLE 18: Remove alias when no longer needed

    cycod alias remove old-alias

  EXAMPLE 19: Update an existing alias

    cycod alias add react-dev "--add-system-prompt \"Updated React expert with Next.js knowledge\" --auto-approve read write" --update
